## 2. Information We Collect

### 2.1 Information You Provide

- **Contact Information**: When you use the "Contact Us" feature, we collect your message and any contact information you voluntarily provide to communicate with you.
- **Search Queries**: Your search queries within the App to improve content discovery and provide relevant results.
- **Content Interactions**: Information about which articles, videos, sounds, and books you interact with, including views, downloads, and engagement metrics.

### 2.2 Automatically Collected Information

- **Device Information**: Device type, operating system version, unique device identifiers, and device settings.
- **Usage Data**: 
  - Pages visited and content accessed
  - Time spent on different sections
  - Download history
  - Playback duration and positions
  - Click-through data
  - Crash reports and error logs

- **Network Information**:
  - IP address (for content delivery optimization)
  - Internet service provider information
  - Browser type and version

- **Local Storage Data**:
  - Theme preferences (dark/light mode)
  - Fullscreen mode settings
  - Cached content and images
  - Download history stored locally on your device

### 2.3 Permissions Required

The App requests the following permissions:

| Permission | Purpose | Platform |
|-----------|---------|----------|
| **INTERNET** | Essential for fetching content from our API server and streaming media | Android, iOS |
| **READ_EXTERNAL_STORAGE** | Access downloaded books, audio, and video files from your device | Android (API < 29) |
| **WRITE_EXTERNAL_STORAGE** | Save downloaded content to your device's storage | Android (API < 29) |
| **Storage Access (iOS)** | Access and manage files in the Files app | iOS |

**Note:** Android 10+ (API 29+) uses the MediaStore API, which doesn't require the legacy storage permissions. The App complies with each platform's storage best practices.

## 6. Data Security

### 6.1 Security Measures

We implement comprehensive security measures to protect your information:

- **HTTPS Encryption**: All data transmitted between the App and our API servers (`https://api.srajalden.com/api`) is encrypted using industry-standard TLS/SSL protocols
- **Secure Communication**: All API requests use secure HTTPS connections exclusively
- **Input Validation**: Strict validation and sanitization of user input
- **Error Handling**: Secure error messages that don't expose sensitive information
- **Logging Security**: Sensitive data is never logged in production environments
- **Regular Updates**: Continuous monitoring and updates of dependencies for security patches

### 6.2 Data Storage

- **Local Storage**: User preferences are stored securely on your device using `SharedPreferences`
- **Server Storage**: Content and usage data are stored on secure servers with access controls
- **Downloaded Content**: Media files are stored in your device's secure storage directories

### 6.3 Limitations

While we implement robust security measures, no method of transmission over the internet or electronic storage is 100% secure. We cannot guarantee absolute security of your information.

### 7.1 Access and Control

You have the right to:

- **Access Your Data**: Request information about what data we hold about you
- **Delete Your Data**: Request deletion of your personal information (except where legally required)
- **Correct Information**: Ask us to correct inaccurate or incomplete information

Within the App, you can:

### 7.3 Device-Level Controls

You can control App permissions through your device settings:

**Android:**
- Settings → Apps → Seraj Al-Din → Permissions

- Toggle permissions on/off as needed

- **Analytics**: While we collect usage data to improve the App, you cannot opt out of essential data collection. However, you can clear the App's cache regularly
- **Contact Communications**: You can unsubscribe from non-essential communications by requesting deletion of your contact information
